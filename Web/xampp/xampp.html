Option #1: Use the command prompt

  1. Press the windows key on your keyboard or press the icon on your task bar.
  2. Type "cmd" and press enter. This will open the command prompt.
  3. You should see "C:\Users\user>"
  4. Move up two folders using "cd../" or until you see "C:\".
  5. Now we want to print a list of programs and the ports they are using.
  6. Type "netstat -ao" and the list will show up.
  7. Each item in this list will have an id associated with it called a "PID". Find "0.0.0.0:3306" and the PID associated with it.
  8. Finally we get to kill the program/task using port 3306 using the PID we found.
  9. Type "taskkill /F /PID" and the PID you found. For my example it would look like "taskkill /F /PID 5864".

Option #2: Change the port number that MySQL is using

  1. Open Xampp and click the config button for MySQL. Open my.ini
  2. Find all instances of "port=3306" and replace it with 3307 or 3308.
  3. Your changes should look similar to this:

    4. [client]
    # password = your_password
    port=3307

    [mysqld]
    port=3307

    # The port the master is listening on.
    # optional - defaults to 3307

  4. Save and close the file. Restarting Xampp couldn't hurt.
  5. There may still be an error showing in red text when you open xampp about port 3306 not being open, so we can change the default MySQL settings for xampp.
  6. Open xampp as an administrator, click on the config button.
  7. Click on Service and Port Settings.
  8. Another window will appear. Click on the MySQL tab.
  9. Change the Main Port to the number you chose; 3307 in my case.
 

MySQL Still Won't Start (Not a port issue)

  1. For this fix, we will be re-installing Xampp.
  2. The first thing we need to do is backup our htdocs folder where our projects are kept. "C:\xampp\htdocs"
  3. Next we can move on to uninstalling xampp. Go to "C:\xampp" and find uninstall.exe
  4. Double click and uninstall xampp.
  5. We need to make sure the xampp folder is empty and/or deleted before re-installing it.
  6. Download the lastest version of xampp from: Apachefriends.org
  7. Go to your downloads folder and double click the setup/install file to begin the re-install.
  8. Make sure to check MySQL, PHP, phpMyAdmin and optionally Mercury Mail Server, and Fake Sendmail.
  9. Hopefully this fixes your issue.

Authentication Issue With phpMyAdmin

  1. If you recently installed xampp and cannot access phpMyAdmin, these steps should help.
  2. Go to "C:\xampp\phpMyAdmin" and open "config.inc.php" in a text editor such as notepad++.
  3. Next you need to find and replace the username and password information.
  4. Here is what it should look like:

    5. /* Authentication type and info */
    $cfg['Servers'][$i]['auth_type'] = 'config';
    $cfg['Servers'][$i]['user'] = 'root';
    $cfg['Servers'][$i]['password'] = 'root';

    /* User for advanced features */
    $cfg['Servers'][$i]['controluser'] = 'root';
    $cfg['Servers'][$i]['controlpass'] = 'root';

  5. You can test if this fixed the issue by opening Xampp and running apache.
  6. Open a browser and go to "localhost/phpmyadmin"
  7. Hopefully this fixed your issue. Happy Coding!

Change Users (From Fresh Install)

  1. Default Authentication Settings for PHPMyAdmin "config.inc.php":

    2. /* Authentication type and info */
    $cfg['Servers'][$i]['auth_type'] = 'config';
    $cfg['Servers'][$i]['user'] = 'root';
    $cfg['Servers'][$i]['password'] = ' ';

    /* User for advanced features */
    $cfg['Servers'][$i]['controluser'] = 'pma';
    $cfg['Servers'][$i]['controlpass'] = ' ';

  2. In PHPMyAdmin click on "User accounts". (Password column will all be set to No initially)
  3. The User account I want to login as when going to PHPMyAdmin is the root user at localhost and once I give it a password our root user will have a "yes" in the password column.
  4. Click on "Change password" tab and enter a new password.
  5. I have chosen "root" as my password.
  6. Click Go.
  7. Now update the "config.inc.php" file to change the user and password.

    8. /* Authentication type and info */

  2. Navigate to your xampp folder "C:\xampp" and find xampp-control.ini, there will probably be less than 20 lines in the file.

Backup Database Files in MySQL Workbench

  1. Click on the Server button in the taskbar and select Data Export.
  2. Select from the list of databases on the left.
  3. The right side will contain a list of tables from that database that you can select from.
  4. There will be a dropdown box near the bottom right of the table list that will allow you to select "Dump Structure and Data".
  5. Export to Dump Project Folder will place files for individual tables into a folder.
  6. Export to Self-Contained File will save all tables to a single file.
  7. Start Export and your database should be backed up.
