PHP Basics

The PHP code (everything between <?php and ?>) will be interpreted by the Web server and converted to normal text before it's sent to your browser. The Web server does all the work, so the Web browser doesn't need to know anything about the php code.

Basic Syntax

A PHP script consists of a series of commands, or statments, each of which is an instruction that the Web server must follow before it can proceed to the next. PHP statements are always terminated by a semicolor (;).


echo("This is a <b>test</b>!");

This statement invokes a built-in function called echo.

Built-in functions are functions that PHP has pre-defined and do not require us to tell them how to work.

Quotes are used to mark the beginning and end of strings of text in PHP.

Parenthesis serve a dual purpose in PHP.

  1. They indicate that echo is a function that you want to call.
  2. They mark the beginning and end of a list of "parameters" you wish to provide.

Variables

Variables: A name given to an imaginary box into which any value may be placed.


$testvariable = 3;

PHP is a loosely typed language. This means that a single variable may contain any type of data (number, string, etc.) and may change types over its lifetime.


// This line will assign a new value to our existing variable and change its type.
$testvariable = "Three";

Operators

The equals sign '=' is called the assignment operator, as it is used to assign values to variables.

Other operators may be used to perform various mathematical operations on values:


$testvariable = 1 + 1; // Assigns a value of 2.
$testvariable = 1 - 1; // Assigns a value of 0.
$testvariable = 2 * 2; // Assigns a value of 4.
$testvariable = 2 / 2; // Assigns a value of 1.

Comments

Comments are a way to describe what your code is doing.


// Tells the PHP interpreter to ignore what comes after.

Concatenation


$testvariable = "Hi " . "there!"; // Assigns a value of "Hi there!".

Variable Interpolation


$var1 = 5; // Assigns a value of 5 to $var1
$var2 = $var1 + 1; // Assigns a value of 6 to $var2

Control Statements

The most basic, and most often-used, control structure is the if-else statement.


if (condition) {
 // Statement(s) to be executed if condition is true.
} else {
 // (Optional) statement(s) to be executed if condition is false.
}

The double-equals (==) used in the condition section is the PHP operator that's used to compare two values to see whether they're equal. (Using a single equals sign would cause the condition to always be true.)

When checking two conditions use the keyword "and".


if ($firstname == "Thirsty" and $lastname == "Penguin") {
 // Do something
}

While Loop

The while loop allows us to use a condition to determine how many times to repeatedly execute a set of statements.


while (condition) {
 // statement(s) to execute as long as the condition remains true.
}

Loops such as this one come in handy when working with long lists.


$count = 1;
while ($count <= 10) {
 echo("$count");
 $count++;
}

  1. Line one creates a variable called $count and assigns it a value of 1.
  2. Line two is the start of the while loop, the condition is (Value of $count is less than or equal to 10).
  3. Line three and four are the body of the while loop, and will be executed over and over, as long as the condition holds true.
  4. Line three outputs the value of $count.
  5. Line four increments (adds 1 to) the value of $count.
    • Increment: $count = $count + 1, $count++
    • Decrement: $count = $count - 1, $count--

For Loop

We have another option for looping through a list of values and that is the for loop.

Here is the same example as above, but implemented as a for loop.


for ($count = 1; $count <= 10; $count++) {
 echo("$count");
}

Multi-Purpose Pages

Once we have a user's name in a variabel on one page, we can pass it with any request to another page by adding the name to the query string of all links:


<a href="newpage.php?name=<?php echo (urlencode($name)); ?>"> A link </a>

There is a shortcut for times when you want to echo a PHP value in the middle of your HTML code.


<a href="newpage.php?name=<?=urlencode($name)?>"> A link </a>

The tags <?=...?> perform the same function as the longer code <?php echo(...); ?>

urlencode:
This function takes any special characters in the string (spaces for example) and converts them into the special codes they need to be in order to appear in the query string.

If $name had a value of Thirsty Penguin, then the output of urlencode would be Thirsty+Penguin and it would be automatically converted back by PHP when the $name variable is created in newpage.php

Say a user enters the site from any other page; they would not have set their name. We will need to add a condition to every page to either show the page's content or prompt the user to enter their name.

The code of a multi-purpose page looks something like this:


<html>
<head>
<title> Multi-Purpose Page Outline </title>
</head>
<body>

<?php if (condition) { ?>
 <!-- HTML content to display if condition is true -->
<?php } else { ?>
 <!-- HTML content to display if condition is false -->
<?php } ?>

</body>
</html>

There is an alternate form of the if-else statement that can make your code more readable in situations like this.


<html>
<head>
<title> Multi-Purpose Page Outline </title>
</head>
<body>

<?php if (condition): ?>
 <!-- HTML content to display if condition is true -->
<?php else: ?>
 <!-- HTML content to display if condition is false -->
<?php endif; ?>

</body>
</html>

We can now look at an example of this in use:


<html>
<head>
<title> Multi-Purpose Page Outline </title>
</head>
<body>

<?php if (!isset($name)): ?>
 <!-- No name has been provided, so we prompt the user for one. -->
 <form action="<?=echo($PHP_SELF)?>" method="get">
  Please enter your name: <input type="text" name="name" />
  <input type="submit" value="GO" />
 </form>
<?php else: ?>
 <p>Your name: <?=$name?></p>
 <p>This paragraph contains a
  <a href="newpage.php?name=<?=urlencode($name)?>">link</a>
  that passes the name variable on to the next document.
 </p>
<?php endif; ?>

</body>
</html>

isset:
This function returns (outputs) a value of true if the variable it is given has been assigned a value (i.e. if a name has been provided) and false if the does not exist (i.e. if a name has not yet been provided).

The exclamation mark (also known as the negation operator, or the not operator reverses the returned value from true to false or vice-versa. Thus, the form is displayed when the $name variable is not set.
$PHP_SELF:
This variable is one of several that PHP gives a default value.

$PHP_SELF will always be set to the URL of the current page. This gives us an easy way to create a form that, when submitted, will load the same page, but this time with the $name variable specified.
 

Database-Driven Website

This is what will happen when someone visits a database-driven web site:

  • The visitor's web browser requests the web page using a standard URL.
  • The web server software (Apache, IIS, etc.) recognizes that the requested file is a PHP script, and so the server interprets the file using a PHP plug-in, before responding to the page request.
  • Certain PHP commands connect to the PHP script.
  • The PHP script stores the content into one or more PHP variables, and then uses the echo function to output the content as part of the web page.
  • The PHP plug-in finishes up by handing a copy of the HTML it has created to the web server.
  • The web server sends the HTML to the web browser as it would a plain HTML file, except that instead of coming directly from an HTML file, the page is the output provided by the PHP plug-in.
 

PHP Forms

As PHP code is activated when a page is requested from the server, user interaction can only occur in a back-and-forth fashion.

The user sends requests to the server, and the server replies with dynamically generated pages.

We can pass variables along with the page request. The variable is passed as part of the "query string", which is the portion of the URL that follows the question mark.

The variables firstname and lastname have the values Thirsty and Penguin respectively.


<a href="welcome.php?firstname=Thirsty&lastname=Penguin">Hi, I'm Thirsty, Penguin!</a>

We can accomplish the same using a form with the GET method.


<form action="welcome.php" method="get">
 <label for="firstname">First name:</label>
 <input type="text" name="firstname">
 <label for="lastname">Last name:</label>
 <input type="text" name="lastname">
 <input type="submit" value="GO">
</form>

In welcome.php we need to add a line that can print the data passed in the query string.


<?php
 echo("Welcome to my website, $_GET[firstname], $_GET[lastname]!");
?>

Using a form with the POST method will not pass information in the query string. However, this will allow you to include large values, or sensitive values (passwords) in the data that's submitted by the form.

If the user bookmarks the resulting page, it will be useless, as it doesn't contain the sumbitted values.


<form action="welcome.php" method="post">...</form>

In the welcome.php page we need to change $_GET to $_POST:


echo("Welcome to my website, $_POST[firstname], $_POST[lastname]!");

 

Connecting to MySQL with PHP

Support for connecting to MySQL is built right into PHP. The following function call establishes the connection:


// Connection Format
mysql_connect(address, username, password);

// Connection Example
$dbcon = mysql_connect("localhost", "root", "my_password");

address
The IP address or hostname of the computer on which the MySQL server software is running "localhost" if it's running on the same computer as the web server software.
username, password
The username and password are the same MySQL username and password used to connect to the MySQL server.

The value returned by mysql_connect is now stored in the variable named $dbcon.

Check Connection

Since MySQL is a separate piece of software, it is possible that the server is unavailable, or inaccessible due to a network outage, or because the username/password combination you provided is not accepted by the server.

In such cases, the mysql_connect function doesn't return a connection identifier. Instead, it returns false. This allows us to react to such failures using an if statement.


@$dbcon = mysql_connect("localhost", "root", "my_password");

if(!$dbcon) {
 echo("<p>Unable to connect to the " .
  "database server at this time.</p>");
 exit();
}

@
Placing an @ symbol in front of the function name tells the function to fail silently if an error occurs, allowing us to display our own error message.
exit()
All this function does is cause PHP to stop reading the page at this point.

Select Database

The next step after establishing a connection is to select a database.


mysql_select_db("database_name", $dbcon);

The $dbcon variable is optional here, mysql_select_db will default to the last connection opened otherwise.

This function will return true if it succeeds and false if an error has occured so we should use another if statement like with mysql_connect.


if(! @mysql_select_db("database_name")) {
 echo("<p>Unable to locate the database_name" .
  "database at this time.</p>");
 exit();
}

With a connection established and a database selected, we are now ready to begin using the data stored in the database.

 

Sending SQL Queries

In the MySQL workbench we can use sql commands and view the results immediately, a similar mechanism exists in PHP; the mysql_query function.


mysql_query(query, connection_id);

SQL

SQL or Structured Query Language, pronunciation "sequel" or "ess-cue-ell". Commands in SQL are also called Queries.

UNIX-based systems will be case-sensitive when it comes to database and table names, as they correspond to directories and files in the MySQL data directory. Otherwise, MySQL is completely case-insensitive.

Table, column, and other names must be spelled exactly the same when they're used more than once in the same command.
