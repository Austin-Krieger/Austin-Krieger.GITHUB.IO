Merging regulation.bin Files

A popular mod to start with is the Grand Merchant mod that lets you buy anything you may need in the game from merchant Kale.

  1. Rather than just extracting the files and blindly moving them all to the Mod Engine 2.0.0 > mod folder, we will start by opening DSMapStudio.


  2. Create a new project, give it a Project Name, make the Project Directory the same folder as the regulation.bin file you just downloaded, and set the Game Executable to eldenring.exe found inside of your Elden Ring > Game folder.


  3. Click Create and when the new project is created the Param Editor tab will be selected.
  4. Now you can type "modified" into the search bars to see the files and values that were editted by the mods creator.


  5. Inside of EquipParamAccessory the mods creator set the sell value of every item to 0. As it happens, this value determines if the item will actually show up in merchant kales' inventory, so you will need to check a weapons sell value whenever merging a new mod.


  6. Type modified into the center search bar to get only the weapons and items that were modified by the mods' creator and select everything you want to extract. Use Ctrl+A to select everything that showed up and then click Edit > Export CSV > To File... > All.
  7. Choose a location to export the file to and name it EquipParamAccessory.csv so you know which param you are importing later.
  8. Follow these same steps for the remaining params: EquipParamGem, EquipParamGoods, ...
  9. Create a new project if one does not exist inside of your Mod Engine 2.0.0 > mod folder and we can start importing the CSV files we just exported.
  10. Search for and select the EquipParamWeapon param and then click Edit > Import CSV From File... All. Do this for the other CSV files you exported and the regulation.bin files are merged. Note: when exporting selected parameters it will only export/import the ones you selected, not the whole file.
  11. Follow these steps for any regulation.bin files you download in the future. The only difference being that you will not need to create a new project inside of your Mod Engine 2.0.0 > mod folder, just open that project and import your new CSV files.
  12. Note: Some mods were made using Yapped-Rune-Bear so if they give you .csv files and you get an error when importing about a number of params mismatch, it's because they used that program and you just need to follow the steps above and make your own .csv files to import.

Merging .dcx (Animation) Files

The most common file to change when merging new weapon moveset mods is the: c0000.anibnd.dcx file.

  1. We will be using Yabber for this tutorial. Setup instructions can be found on the Getting Started page.
  2. Drag the downloaded chr > c0000.anibnd.dcx file onto Yabber.exe to unpack the file.


  3. Assuming you already have a chr > c0000.anibnd.dcx file in your Mod Engine 2.0.0 > mod folder, you will need to unpack that as well.
  4. Hopefully the creator left merging instructions in their mod description or readme.txt file listing the aXX.tae files you need to replace.
  5. Navigate to the tae folder.
  6. Select the new .tae files and replace them in your main mods' c0000.anibnd.dcx folder that we unpacked.
  7. Next you will drag the main mods unpacked c0000.anibnd.dcx folder onto Yabber.exe to repack the file and begin testing your new mod.
  8. You can follow the same process when merging FXR files (sfx changes).
