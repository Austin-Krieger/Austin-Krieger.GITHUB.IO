Start With the Basics

Here we will cover some of the basics for making your own mods.

  1. Find the mods folder for Don't Starve Together.

    2. D:\Steam\steamapps\common\Don't Starve Together\mods

  2. Create a new folder for your mod where you will place your mod files.
  3. Create a new file inside this folder with the name modinfo.lua. This will store the information about your mod.
  4. Create another file named modmain.lua. This is where your mods' functionality will be set.
 

Mod Info

Configuration Options

  1. The name of the mod.

    2. name = "..."

  2. The description of the mod.

    3. -- Single line
    description = "..."

    -- Multiline
    description = [[
     Show a multiline description in game.
     Width: 52 characters
     Height: 17 rows
    ]]

  3. Who made the mod.

    4. author = "..."

  4. The current version of the mod.

    5. version = "1.0"

  5. The game needs to know what API your mod expects. This should be set to the current version.

    6. api_version = 10

  6. This is the URL name of the mod's thread on the forum.

    7. forumthread = "http://forums.kleientertainment.com/..."

  7. Specify a custom icon for this mod.

    8. icon_atlas = "modicon.xml"
    icon = "modicon.tex"

  8. Let the mod system know this mod is functional with Don't Starve Together (Select where your mod should work).

    9. dst_compatible = true
    dont_starve_compatible = false
    reign_of_giants_compatible = false
    hamlet_compatible = false
    forge_compatible = false

  9. Let clients know they need the mod in order to join the server. This can be set to false if it won't be noticeable to other players (UI Changes).

    10. all_clients_require_mod = true

  10. This mod does not need to be listed in the server's mod listing.

    11. client_only_mod = false

  11. Other players can use these filter tags to find a server with this mod installed.

    12. server_filter_tags = { "Filter Tag", " ", " ", ... }

  12. Put it all together:

    13. name = "Mod Name"
    description = "Mod Description"
    author = "Your Name"
    version = "1.0"
    api_version = 10
    forumthread = " "

    icon_atlas = "modicon.xml"
    icon = "modicon.tex"

    dst_compatible = true
    dont_starve_compatible = false
    reign_of_giants_compatible = false
    hamlet_compatible = false
    forge_compatible = false

    all_clients_require_mod = true
    client_only_mod = false

    server_filter_tags = { "Filter Tag" }

  13. You can also add configuration options that can be accessed by your scripts.

    14. configuration_options = {
     name = " ", -- Unique name for script access.
     label = " ", -- Title for the player to see (27 char max).
     hover = " ", -- Hint text on mouse over. (156 char max)

     options = {
      {
       description = " ", -- Short string shown to the user. (14 char max)
       data = " " -- string, number or boolean value.
      }
     },
     default = 1
    }

How to make a title in options:

{
 name = "someName", -- Any unique name
 label = "yourTitle",
 hover = "on mouse hover hint",
 options = { { description = " ", data = false}}, -- One item with a boolean option and no description
 default = false
}

How to access options from scripts:

GetModConfigData("optionName")

Mod Image

  1. Using the TEXTool program we can convert .png to and from the .tex format.
  2. Download: TEXTool.zip
  3. Convert your .png to a .tex file and place it in the folder for your mod.

    4. D:\Steam\steamapps\common\Don't Starve Together\mods\Mod Name

  4. Create a new file in your mods' folder called modicon.xml. Make sure this matches with the name you used in modinfo.lua.

    5. <Atlas>
     <Texture filename="modicon.tex" />
     <Elements>
      <Element name="modicon.tex" u1="0.001" u2="0.999" v2="0.999" v1="0.001" variant="" />
     </Elements>
    </Atlas>

 

Uploading to Steam Workshop

  1. Inside of Steam, search through your tools and find the Don't Starve Mod Tools.
  2. Install and launch the program.
  3. Change the Target Game's Workshop to Don't Starve Together.
  4. Browse for and select the folder for your new mod.
  5. Publish!
