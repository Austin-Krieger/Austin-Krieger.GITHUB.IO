Setup Zombie Spawner

  1. In the Entity Browser Pane search for spawner_zm_factory.
  2. Click and drag it into the map. If you have the extra assets dlc installed, you will see the model of a zombie.
  3. With the entity selected (Shift + Click) go to the Entity Info Pane.
  4. First we need to change the coop_count from 1 to 9999.
  5. Change ALTERONSPAWN to False.
  6. Change MAKEROOM to True.
  7. Next we need to hit Add KVP to add some User-Defined properties.
  8. Click Add KVP

    9. Property / Key: script_disable_bleeder
    Value: 1

  9. Click Add KVP

    10. Property / Key: script_forcespawn
    Value: 1

  10. Click Add KVP

    11. Property / Key: script_noteworthy
    Value: zombie_spawner

Spawn Zombies (Map)

Now we need to add locations for the zombies to spawn.

  1. Right Click in the Game or Grid view and select script → struct.
  2. With our new struct selected, go to the Entity Info Pane (N) and change the targetname to the name of the current zone.

    3. targetname: start_zone_spawners

  3. Click Add KVP.

    4. Property / Key: script_noteworthy
    Value: riser_location

  4. Click Add KVP.

    5. Property / Key: script_string
    Value: find_flesh

  5. Bonus: You can use Space Bar to make a copy of the selected struct near the original.

Spawn Zombies (Window)

  1. Right Click in the Game or Grid view and select Misc → prefab.
  2. Double click zm > zm_core and search for barricade_receiver_wood.map and hit Open.
  3. Build up walls around the window like so: (Refer to the Walls section above)
  4. Remember we can use Middle Mouse Click while the walls are selected to apply another texture present in the map or Double Click a texture from the Textures Pane.
  5. Deselect the walls (Esc).
  6. With the window selected hit Add KVP. The Value for this one must be unique on your map or the zombies will not know what to do when they spawn.

    7. Property / Key: script_string
    Value: receiver_set_entry_b

  7. Deselect the window (Esc).
  8. Right Click in the Game or Grid view and select script > struct. Move the struct entity into the window room.
  9. With our new struct selected, go to the Entity Info Pane (N) and change the targetname to the name of the current zone.

    10. targetname: start_zone_spawners

  10. Click Add KVP.

    11. Property / Key: script_noteworthy
    Value: riser_location

  11. Click Add KVP. The Value for this one should match the window.

    12. Property / Key: script_string
    Value: receiver_set_entry_b

 

Spawning Dogs

In the default map they include a spawn point, but no way for the dogs to actually spawn. The following steps will fix that issue.

  1. First we need to go to the Entity Browser Pane and search for Actor > Spawner_Zm_Factory > Zombie_dog.
  2. Drag one onto the map, placing it where the player won't accidentally run into it (collision issues).
  3. Select the dog and change the Entity Info. You can do the same to the default spawn point added when you made the map (targetname should be unique).

    4. Property: ALTERONSPAWN
    Value: False


    Property: MAKEROOM
    Value: True


    targetname: zombie_dog_spawner

  4. Click Add KVP

    5. Property / Key: script_forcespawn
    Value: 1

  5. Click Add KVP

    6. Property / Key: script_noteworthy
    Value: zombie_dog_spawner

  6. Next we need to make a spawn point (or copy an existing one).
  7. Right Click in the Game or Grid view and select script → struct.
  8. Line up the struct entity so that the center line, lines up with the floor.
  9. With our struct selected, go to the Entity Info Pane (N) and change the targetname to the name of the current zone.

    10. targetname: test_zone_spawners

  10. Click Add KVP

    11. Property / Key: script_noteworthy
    Value: dog_location

  11. Dogs will now spawn when they should.

  5. Open your Maps' .gsc file from the usermaps\zm_tutorial_level\scripts\zm folder.
  6. Inside the main() function add the following lines:

    7. // Brutus
    brutus::init();

  7. Scroll up to the line #using scripts\zm\zm_usermap; and paste the following lines after it:

    8. // Brutus
    #using scripts\custom\brutus;

  8. Now we need to open our Maps' zone file zm_tutorial_level.zone from the zone_source folder and add the following lines at the end:

    9. // Brutus
    scriptparsetree,scripts/custom/brutus.gsc
    scriptbundle,brutus_bundle
    xmodel,brutus_helmet
    xmodel,bo2_brutus_fb_death
    xmodel,perk_clip
    fx,Brutus/spawn_fx
    fx,Brutus/chest_fx
    fx,Brutus/helmet_smoke
    fx,fire/fx_fire_ground_rubble_50x50

  9. There are some customizable settings near the top of the brutus.gsc file we just added. We can change his base health, max rounds before spawning and so on.
  10. Now we need to open our map in Radiant and place all of the .map files you just downloaded.
  11. For each of the perk_lock.map files, you will need to line up the red (crosshair like) box with the red box of the respective perk.
  12. For each mystery box on your map, drag in a box_lock.map prefab and place it with the green box in the front of the box but not quite touching it and the red boxes should be inside the box prefab and halfway inside the ground.
  13. Place one brutus_spawner.map prefab anywhere in your map.
  14. Place the brutus_start_zone.map prefab in the starting zone of your map. If you changed the name of this zone then Right Click > Prefab > Stamp Prefab and change the script_string value to the name of your zone.
  15. Do the same when placing the brutus_your_zone.map file in other zones.
