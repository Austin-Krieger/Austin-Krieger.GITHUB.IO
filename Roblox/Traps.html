Example Axe Trap

In Roblox Studio, click the New button in the upper-left or use an existing project.

Prototype axe trap built from simple parts.

  1. Start by adding a spawn location to the level.
  2. Create a platform for players to run on.
  3. Create a frame for the trap and group the parts together. (Anchor non-moving parts)
  4. Add simple parts to make it look like a swinging axe.
  5. Group the parts and drag the group into the trap frame group.
  6. Create a hinge constraint between the trap base and the top of our swinging axe.
  7. Click on the new hinge constraint object and set some values.
  8. ActuatorType: Servo
  9. AngularSpeed: 5 (Change it as needed)
  10. ServoMaxTorque: 40000 (More torque is required for "Heavier objects")
  11. TargetAngle: 90 (Our script with change this value)
  12. Click the circle + button on the servo object and search for and add a script. (We want to make sure the axe keeps swinging)

    13. -- Link the trapPart variable to its parent object.
    local trapPart = script.Parent

    local function swingTrap()
     wait(.1)
     if trapPart.HingeConstraint.CurrentAngle >= 15 then
      trapPart.HingeConstraint.TargetAngle = -45
     end
     if trapPart.HingeConstraint.CurrentAngle <= -15 then
      trapPart.HingeConstraint.TargetAngle = 45
     end
     swingTrap()
    end

    swingTrap()

 

Hinged Platform

  1. Start by adding a small block as a base for our hinged platform and anchor it in place.
  2. Create the platform that will tip back and forth. Make sure it's centered left-to-right with the support part so that the final assembly balances. Also consider applying a different color so that players recognize it's not a typical platform.

    3. Remember not to anchor this part since it is meant to move.

  3. Rename the parts PlatformSupport and HingePlatform.
  4. Next we need to attach the parts with a hinge constraint.
  5. Change the mass of the HingePlatform.

    6. In Roblox, a part’s mass is internally calculated based on its size and density. If the hinged platform’s mass is too low, it will move too quickly and drop players off into empty space before they can reach the other side. If its mass is too high, the platform will move too slowly and players will just walk across it like a normal anchored part.

  6. Select the HingePlatform in the game editor window, or select it from the Explorer window.
  7. In the Properties window, find the Part section and look for a row named CustomPhysicalProperties (if necessary, expand the Part section by clicking the small arrow next to its name).
  8. Press the play button to test. Adjust the platform's mass (density) so that players are challenged to reach the other side quickly.
 

Turning Platform

  1. Create a base and anchor it in place
  2. Create the platform that will eventually spin around and around. Position this platform so that it sits on top of the base we just created.
  3. Rename the parts PlatformBase and TurnPlatform (TurningPlatform also acceptable).
  4. Add a hinge constraint. Optionally change the platform color.
  5. Make the platform spin. In the Explorer window, find and expand the branches for both PlatformBase and TurnPlatform.
  6. Select the HingeConstraint object and locate the ActuatorType property. Click in the area to its right and select motor.
  7. To make the platform turn, type in a new value for the AngularVelocity property that appeared. This value determines how fast the motor should turn (A value of 0.5 should be fine).

    8. To change the direction of the platform, just change the value of the AngularVelocity to a negative number.

  8. Finally, change the value of MotorMaxTorque to 10000. This value may need to be even higher for a motor to turn a very large/heavy part.

    9. Now press the Play button to test. If you did everything right, the turning platform should slowly spin around on the base. The player’s challenge is to jump on the platform at the correct time, then let it carry them around until they can safely jump off to another platform.

 

Dangerous Obstacle

  1. Create a platform for the players to walk on and a spawn location for playtesting.
  2. Create a new part and position it on top of the platform, leaving enough space for the players to jump over the trap without touching it.
  3. Anchor the parts in place so that they can't slide off or fall into the void.
  4. Lets change the color of the obstacle to something scary (red).
  5. Rename the parts in the Explorer window. Platform and Trap are simple and effective here.
  6. At this point touching the trap will do nothing. We need to add a script to make it kill anyone that accidentally lands on it.
  7. Hover over Trap in the Explorer window and click the circle + icon, search for script and click to add.
  8. New scripts will open in a new tab just above the game editor window. Delete the default hello world line.

    9. local trapPart = script.Parent

    local function onPartTouch()
     local partParent = otherPart.Parent
     local humanoid = partParent:FindFirstChildWhichIsA("Humanoid")
     if humanoid then
      -- Set player's health to 0
      humanoid.Health = 0
     end
    end

    trapPart.Touched:Connect(onPartTouch)

  9. Press the Play button. Walk to and touch the trap. If you did everything right, your player should instantly explode.

Explaining the Code

  1. The first line just sets a variable that tells the script what specific game part it’s linked to. In this case it’s linked to the trap part, the “parent” of the script.
    2. local trapPart = script.Parent
  2. The next part, is a function. Functions are sets of related steps which can be used over and over. This function checks if a careless player (not another random part in the world) touched the trap. If so, it sets the player's health to 0 which instantly destroys them.
    3. local function onPartTouch(otherPart)...
  3. The final line connects the trap and the function with a Touched event. That makes the function run whenever the trap part is touched.
    4. trapPart.Touched:Connect(onPartTouch)
 

Jump Boost

  1. Create a new part (sphere), positioned on top of a walking platform and change its color.
  2. Anchor the part in place so that it doesn't roll off the platform.
  3. Rename the parts Platform and JumpBoost.
  4. In the Explorer window, hover over the part name and click the circle + button. Search for the script option and click on it.
  5. The new script will open in a new tab. Replace the default line with the following.

    6. local boostPart = script.Parent
    -- Set a variable for boosted jump power
    local BOOSTED_JUMP_POWER = 125

    local function onPartTouch()
     local partParent = otherPart.Parent
     local humanoid = partParent:FindFirstChildWhichIsA("Humanoid")
     if humanoid then
      boostPart.CanCollide = false
      local currentJumpPower = humanoid.JumpPower
      if currentJumpPower < BOOSTED_JUMP_POWER then
       humanoid.JumpPower = BOOSTED_JUMP_POWER
       wait(10)
       humanoid.JumpPower = currentJumpPower
      end
     end
    end

    boostPart.Touched:Connect(onPartTouch)

  6. Press the Play button and test out the new jump boosting pickup.

Explaining the Code

  1. The first line just sets a variable that tells the script what specific game part it’s linked to. In this case it’s linked to the trap part, the “parent” of the script.
    2. local trapPart = script.Parent
  2. The next part creates a new variable with a value of 125. Later in the script, this variable will be used to increase the player's jump power when they touch the boost item.
    3. local boostPart = script.Parent
    local BOOSTED_JUMP_POWER = 125
  3. "boostPart.CanCollide = false" makes the JumpBoost part into a "ghost" object only for players. This lets players touch the boost but also walk through it wihtout being forced to step over it.
  4. "local currentJumpPower = humanoid.JumpPower" sets a variable to the player's current jump power so that the script has a reference to the default jump power.
  5. "if currentJumpPower < BOOSTED_JUMP_POWER then" checks to make sure the player's current jump power is less than the boosted jump power value. This ensures that the player can only gain a jump boost if they're not already boosted.
  6. "humanoid.JumpPower = BOOSTED_JUMP_POWER" increases the player's jump power to the value of BOOSTED_JUMP_POWER
  7. "wait(10)" makes the code wait 10 seconds before continuing.
  8. "humanoid.JumpPower = currentJumpPower" sets the jump power of the player back to the default.
  9. The final line connects the boost part and the function with a Touched event.
    10. boostPart.Touched:Connect(onPartTouch)

Removing Pickups

  1. Adding "boostPart:Destroy()" just before the wait(10) line will destroy the object. This works well if you want a pickup to be used only once.
  2. If you want the JumpBoost to respawn after a player has picked it up then we can change the parent of the object and change it back after the 10 seconds are up.

    3. local boostPart = script.Parent
    -- Set a variable for boosted jump power
    local BOOSTED_JUMP_POWER = 125

    local function onPartTouch()
     local partParent = otherPart.Parent
     local humanoid = partParent:FindFirstChildWhichIsA("Humanoid")
     if humanoid then
      boostPart.CanCollide = false
      local currentJumpPower = humanoid.JumpPower
      if currentJumpPower < BOOSTED_JUMP_POWER then
       humanoid.JumpPower = BOOSTED_JUMP_POWER
       -- boostPart:Destroy()
       boostPart.Parent = nil
       wait(10)
       boostPart.Parent = workspace
       humanoid.JumpPower = currentJumpPower
      end
     end
    end

    boostPart.Touched:Connect(onPartTouch)

  3. Play test and enjoy a respawning jump booster.
